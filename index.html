Your Learning Roadmap

Five phases, roughly in order. Each one builds on the last. Click any phase to expand it — and don't rush. Understanding beats speed every time.

0
PHASE 00
Learn to Think in Code
⏱ Weeks 1–2
▾
⚠️
Don't open Unity yet. The most common reason beginners quit is jumping into Unity with zero coding knowledge. There's too much happening at once. Two weeks here saves months of frustration.

You need to understand three fundamental ideas before anything else makes sense:

✓
What a variable is
A named box that holds information — like playerHealth = 100
✓
What a loop does
Doing something over and over — like checking if the player is alive every frame
✓
What a function is
A named chunk of instructions you can reuse — like Jump() or TakeDamage()
✓
Watch the first 2 weeks of CS50 (free, Harvard)
cs50.harvard.edu/x — genuinely the best free intro to programming in the world
✓
Look up some basic C# syntax
That's the language Unity uses. Just enough to recognise it — you'll learn it properly in Unity
1
PHASE 01
Set Up Unity
📅 Day 1 of Week 3
▾

This only takes an hour. Don't overthink it.

✓
Download Unity Hub from unity.com
Free. It manages your Unity versions and projects
✓
Install the LTS version of Unity
LTS = Long Term Support. Most stable, most tutorials use it. Don't install the latest "Tech" version
✓
Create a new project using the 2D Core template
2D is simpler to start — fewer moving parts. You can always go 3D later once you've got the hang of things
✓
Install Visual Studio Community
Unity will actually prompt you to do this. It's the code editor that plugs directly into Unity — free
✓
Click around and break things
Seriously. Just poke everything. Nothing you do here will permanently break anything
2
PHASE 02
Learn Unity's Five Fundamentals
⏱ Weeks 3–5
▾

Learn these one at a time. Once they click, Unity makes complete sense.

✓
🖥️ The Interface — Scene, Game, Hierarchy, Inspector
These four windows are the cockpit of Unity. Learn what each one is for
✓
📦 GameObjects & Components
Everything in Unity is a GameObject (a container) with Components stuck on it (behaviours). Once this clicks, everything else makes sense
✓
📐 The Transform Component
Controls position, rotation, and size of everything in your scene. You'll use this constantly
✓
🧩 Prefabs
Reusable, saveable GameObjects. Like a template you can stamp out as many times as you want — essential for enemies, coins, bullets
✓
⚡ Basic Physics — Rigidbody2D and Collider2D
These two components give objects gravity and the ability to bump into each other. Most games need them
3
PHASE 03
Write Your First Scripts
⏱ Weeks 4–6
▾

C# scripting is where things start feeling like magic. Focus on just these things first — nothing more.

💡
Tip: Don't try to memorise syntax. Look things up constantly. Every professional developer does. The skill is knowing what to search for, not memorising everything.
✓
⚙️ Start() and Update()
Start runs once when the object appears. Update runs every single frame. These two methods are the heartbeat of every Unity script
✓
📝 Variables and types: int, float, bool
int = whole numbers, float = decimal numbers, bool = true/false. That's most of what you'll use
✓
🔀 if statements
"If the player presses space, jump." The basis of virtually all game logic
✓
🕹️ Input.GetKey() and Input.GetAxis()
This is how you detect keyboard input. Once you know this, you can make things move — instant satisfaction
✓
🐛 Debug.Log()
Prints a message to Unity's console. Your absolute best friend when something isn't working. Use it constantly
4
PHASE 04
Build Your First Game
⏱ Weeks 6–10
▾
🏆
The golden rule: Start tiny. The #1 beginner mistake is trying to build an open-world RPG as project one. It always ends in abandonment. A finished bad small game beats an abandoned great big one — always.

Best first projects for an explorer:

✓
🐦 Flappy Bird clone — the classic beginner project
Teaches: movement, gravity, collision, score, game over screen. A complete game loop in one tiny project
✓
⭕ Top-down coin collector
Teaches: movement in all directions, triggers, UI, timers. Great for exploring what genre you enjoy
✓
💣 Simple Breakout / Pong clone
Teaches: bouncing physics, win/lose conditions, ball mechanics. Very satisfying to play once done
✓
🎮 Share it when done — even if it's rough
Show family, friends, anyone. The reaction to "I made this" is the best motivation fuel there is
✓
🚀 Optional: publish it on itch.io for free
itch.io is a free, friendly platform for indie games. No gatekeeping. Great first milestone
Discover Your Direction
What Kind of Game?

You're exploring right now — and that's great. Tap a type to see what it involves, so you can start narrowing down what excites you most.

⚔️
2D Platformer / RPG
Side-scrolling action, pixel art, classic feel
🔫
3D Action / Shooter
Three-dimensional worlds, movement, combat
🧩
Puzzle / Strategy
Brain-teasers, systems, satisfying logic
🌍
Open World
Exploration, freedom, big environments

⚔️ 2D Platformer / RPG Path

  • Unity's 2D Core template is perfect — start here
  • Learn Tilemaps to build levels from tile sheets
  • Learn Rigidbody2D for gravity and jumping physics
  • Explore the Sprite Editor for character animations
  • Great first game: a simple Mario-style platformer with 3 levels
  • Free art assets: itch.io/game-assets or Kenney.nl

🔫 3D Action / Shooter Path

  • Start with the 3D Core template in Unity
  • Unity has an official beginner tutorial: Roll-a-Ball — do this first
  • Learn the CharacterController component for 3D movement
  • Learn Cinemachine for third-person or first-person cameras
  • Great first game: a simple ball-rolling or collect-the-cubes game
  • 3D is harder to start than 2D — but totally doable with patience

🧩 Puzzle / Strategy Path

  • Either 2D or 3D works — puzzle games often use 2D for simplicity
  • Focus on game logic and state management in your scripts
  • Learn how to store and compare data (e.g. "is this tile matched?")
  • Great first game: a simple Match-3 (like Candy Crush) or Sokoban clone
  • Puzzle games rely heavily on code logic — great for learning C#
  • Less art required — you can use shapes and colours early on

🌍 Open World — A Note of Caution

  • Open world is the hardest genre to build — even for pros
  • The dream is totally valid — but don't start here as project one
  • Start with a contained 3D game first (e.g. Roll-a-Ball, simple platformer)
  • Learn NavMesh for AI pathfinding, Terrain for landscapes
  • Learn Procedural generation later — it's powerful but complex
  • By the time you've built 2–3 small games, open world becomes achievable

Staying In The Game
Tips for Actually Sticking With It

Most guides skip this part. It's the most important section. Motivation is a skill, not a feeling.

📋
Keep a "Tiny Wins" List
Every time something works — even "the box moves now" — write it down. Momentum is everything in the early weeks. Reading back your wins on tough days actually works.
👥
Share Early, Share Often
Show a bouncing square to a friend or family member. The reaction to "I made this" is the most powerful motivation fuel there is. Don't wait until it's polished.
🐛
Bugs Are Normal
Every developer at every level spends significant time confused. It's the job, not a sign you're failing. The skill isn't knowing everything — it's knowing how to find the answer.
📈
Compare to Last Week, Not Others
Don't compare your project to finished games on Steam. Compare where you are now to where you were seven days ago. That's the only meaningful measure of progress.
🚫
Resist Scope Creep
"What if I added a crafting system... and a story... and multiplayer..." — everyone does this. Every idea is valid for project three. Right now: finish the small thing first.
💥
Break Things on Purpose
Change random numbers. Delete components. See what explodes. This is how tinkerers learn fastest and it's genuinely fun. Nothing in a learning project is irreplaceable.
